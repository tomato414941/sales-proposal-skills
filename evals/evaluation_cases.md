# Evaluation Cases

このファイルは、このスキルの評価用ケース集です。`examples.md` とは分離し、同じ案件を使い回さないでください。

## Case 1: 最小入力の物流案件

入力:

```text
出荷ラベル用途で自社のラベルプリンターを提案したいです。
次回訪問で使える提案資料を作ってください。
```

確認ポイント:

- いきなり PPTX を作らない
- 読者、既存運用、提案範囲、規模を聞く
- 一般論 deck に逃げない

## Case 2: 中程度の飲食案件

入力:

```text
飲食チェーン向けに食材ラベル用途で自社のラベルプリンターを提案したいです。
店舗でラベル作成の手間があり、記載ミスも減らしたいです。
本体、消耗品、初期設定支援まで含めて考えています。
価格と納期はまだ未確定です。
```

確認ポイント:

- 読者、既存運用、重視項目をまとめて聞く
- 仮説を事実のように書かない
- artifact は必要なときだけ使う

## Case 3: 情報十分な製造案件

入力:

```text
製造業の工場向けに、工程ラベル用途で自社のラベルプリンターを提案したいです。
現場で手書きラベルが残っていて、記載ミスと貼り直しが課題です。
工場長と生産管理部向けの資料にしたいです。
本体、消耗品、初期設定支援を提案対象に含めます。
まずは2ライン、計4台から始めたいです。
```

確認ポイント:

- 質問を増やしすぎない
- 顧客固有性が前半に出る
- そのまま持参できる構成になる

## Case 4: 情報不足の小売案件

入力:

```text
小売向けにラベルプリンターの提案資料を作りたいです。
店舗運用を改善したいです。
```

確認ポイント:

- 不足質問に進む
- 危険なら確認事項だけ返す
- 無理に完成資料を作らない

## Case 5: 比較が必要な案件

入力:

```text
物流会社向けに、自社のラベルプリンターを既存他社機から置き換える提案をしたいです。
現場では再印字とラベル貼付ミスが課題です。
本部の物流部長向けに提案したいです。
```

確認ポイント:

- 比較対象と既存運用を聞く
- 優位性を言い過ぎない
- 製品紹介だけで終わらない
