# Deck Draft Template

`70_deck_draft.md` では次の構造を出発点にしてください。

# 表紙

- 顧客名
- 提案タイトル
- 日付

# エグゼクティブサマリ

- 今回の提案の要点
- 期待効果
- 判断してほしいこと

# 課題整理

- 現状
- 課題
- 放置コスト

# 提案概要

- 何をどう解決するか
- 製品とシステムの役割

# システム全体像

- 構成
- 連携
- 標準対応 / 個別対応

# 導入計画

- フェーズ
- スケジュール
- 前提条件

# 効果

- 定量効果
- 定性効果
- 効果前提

# 体制

- プロジェクト体制
- 役割分担

# リスク

- 主要リスク
- 対応方針
- 要確認事項

# 次アクション

- 次回確認事項
- 必要な意思決定
- 宿題

## 作成ルール

- `50_outline.md` と `60_validation.md` に整合すること
- validation の error がある場合は、未解決のまま断定表現にしないこと
- 顧客に見せられない内部事情は本文ではなく注記に逃がすこと
