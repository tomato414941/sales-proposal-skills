# Examples

このファイルは、営業担当者が実際に入力しそうな自然文と、それに対してこのスキルがどう質問し、どう資料化するかを見るための例です。

## 例1: 最小入力から質問に進む

入力:

```text
バーガーキング向けに、食材ラベル用途で自社のラベルプリンターを提案したいです。
次回訪問で使える提案資料を作ってください。
```

最初に整理する骨格:

- 顧客: バーガーキング
- 商材: ラベルプリンター
- 用途: 食材ラベル
- 目的: 次回訪問用の提案資料

返す追加質問の例:

- 店舗で今どのように食材ラベルを作っていますか
- 誰に見せる資料ですか。店舗責任者向けか、本部向けか、両方ですか
- 本体だけでなく、消耗品や導入支援も含めて提案しますか
- まず何店舗くらいから始める想定ですか

その後の流れ:

- 必要なら骨格を artifact で見せて方向確認する
- 回答を反映して提案の勝ち筋を決める
- スライド構成を作る
- 持参可能な PPTX を返す

## 例2: 中程度の入力から最小質問で進む

入力:

```text
バーガーキング向けに、食材ラベル用途で自社のラベルプリンターを提案したいです。
店舗でラベル作成の手間がかかっていて、手書き運用による記載ミスもありそうです。
本体、消耗品、初期設定支援まで含めて考えています。
価格や納期はまだ未確定なので断定しないでください。
```

最初に整理する骨格:

- 顧客: バーガーキング
- 商材: 本体、消耗品、初期設定支援
- 課題: ラベル作成の手間、記載ミス
- 制約: 価格と納期は未確定

返す追加質問の例:

- 誰に見せる資料ですか
- 現在のラベル運用は手書き中心ですか、それとも既存プリンターがありますか
- 最初は何店舗、何台程度を想定していますか

その後の流れ:

- 読者と既存運用を確定してからスライド構成に入る
- 情報量が多ければ slide plan を artifact で確認してから進む
- 効果と導入イメージを顧客向けに当て込む

## 例3: 情報が揃っていてそのまま資料化する

入力:

```text
バーガーキング向けに、食材ラベル用途で自社のラベルプリンターを提案したいです。
店舗で手書きラベル運用が残っていて、仕込み日時と消費期限の記載ミスが課題です。
店長と店舗運営部向けの資料にしたいです。
本体、消耗品、初期設定支援を提案対象に含めます。
大掛かりなシステム改修は避けたいです。
次回訪問でそのまま使える提案資料にしてください。
```

期待する流れ:

- 追加質問は最小限またはゼロ
- artifact は不要なら出さない
- 読者と課題に合わせて構成を作る
- 持参可能な PPTX を返す

## 例4: 仮説前提でたたき台にする

入力:

```text
ラベルプリンターの提案資料を作りたいです。
飲食チェーン向けで、店舗運用を改善したいです。
詳しい運用はまだ分かっていません。
```

返す追加質問の例:

- 食材ラベル、値引きラベル、出荷ラベルなど、どの用途を想定していますか
- 誰に見せる資料ですか
- 本体だけの提案か、消耗品や支援も含めますか

回答が十分でない場合の扱い:

- 一般論で断定しない
- チャットで `現時点の情報に基づくたたき台です` と明示する
- 必要なら骨格だけ artifact で見せて前提を握る
- 未確認前提を伝えたうえで資料化するか、必要なら確認事項だけ返す
