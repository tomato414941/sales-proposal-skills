# sales-proposal-skills

営業担当者の少ない入力から提案に必要な情報を整理し、不足情報を回収しながら、最終的に組み込みの PPTX 機能で顧客向け提案資料を作る skill です。

価値の中心は、AI でスライドを作ること自体ではなく、次の4点です。

- 少ない入力から前に進めること
- 顧客ごとに当て込まれた提案にすること
- 言い過ぎや誤読を防ぐこと
- 最後にそのまま持参できる PPTX まで仕上げること

## 前提条件

- Claude の skill 実行環境
- 組み込み `pptx` スキルが利用できること

## 使い方

案件情報、議事録、RFP、製品情報などを渡して、提案資料を作りたいと伝えてください。

この skill は次の順で進みます。

1. 入力から提案の骨格を整理
2. スライド作成前に、不足情報をまとめて確認
3. 必要なら中間確認として artifact で骨格や構成を確認
4. 提案戦略とスライド構成を設計
5. 内容を批判的に見直し
6. 最後に PPTX を生成
7. 生成後に持参品質を確認

主な成果物:

- 追加確認が必要な場合は、その確認事項
- 仮説前提のたたき台
- 提案戦略
- スライド構成
- 最終 PPTX

読者、既存運用、提案範囲などの必須確認項目が埋まっていない場合は、PPTX を作らず確認に戻ります。利用可能なら質問ツールで確認し、使えない場合は通常の会話で同じ確認を返します。

Claude Code では、必要に応じて `.proposal/` に途中成果物を残せますが、必須ではありません。

artifact は主成果物ではなく、営業担当者と AI が提案の骨格や slide plan に合意するための補助表示として必要な場合だけ使います。

examples は使い方の説明用です。評価には [evaluation_cases.md](evals/evaluation_cases.md) のような別ケース群を使い、同じ案件で過剰最適化しないようにします。
